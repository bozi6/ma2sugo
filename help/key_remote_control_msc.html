MIDI Show Vezérlés (MSC)

MIDI Show Control (MSC) egy másik módja a rendszer vezérlésének.

1991-ben jelent meg a MIDI protokoll kiterjesztéseként. A grandMA2 képes fogadni és küldeni is az MSC-t.

Az MSC-nek sok beállítása van. Ezek többésge úgy van beállítva, hogy megfeleljen az adó és vevõkészülékeknek.

A beállításoknál akad némi leírás az MSC koncepciójáról és néhány szó az adatátvitelrõl hálózaton keresztül MIDI használata hálózaton.

 

Az MSC beállítása

A beállítások eléréséhez, nyomd meg a Setup gombot és utána a Midi Show Control gombot a Console fülön.

A beállítások így néznek ki:

Négy sor van különbözõ beállításokkal és két ellenõrzõ mezõ.

Az ellenõrzõ mezõk mutatják a bejövõ, és kimenõ MSC-t. Ezek értelmezhetõ adatokat mutatnak, ami azt jelenti, hogy nem mutatják a nyers hexadecimális adatokat..

A fölsõ sorban kér bemeneti mezõ van és egy kapcsológomb:

  • Channel MIDI In:
    A MIDI szabványvban 16 különbözõ csatorna áll rendelkezésre. Az itt lévõ számnak stimmelnie kell a küldõ eszköz beállításával.
  • Channel MIDI Out:
    Ennek a számnak, meg stimmelnie kell a fogadó MSC eszköz számával.
  • MIDI Thru:
    Ez a kapcsoló, állítja be, hogy a bejövõ MIDI jel, továbbítva legyen a MIDI kijáratra is..
Fontos:
Ha az MSC parncsok ugyanazon a MIDI csatornán lesznek küldve és fogadva is, akkor egy hurok jön létre.

A következõ két sor hasonló, az egyik a bejövõ, a másik pedig a kimenõ MSC-t mutatja.

  • Eszközök:
    112 különbözõ eszköz van az MSC-ben. Van továbbá egy "All" lehtõség is. Ezt a Send to gombbal lehet beállítáani a bal alsó sarokban - lejjebb olvashatsz róla. Ezek a bemenõ értékek 0 tól 111-ig lehetnek.
  • Csoport:
    Az MSc szabvány megengedi, hogy az eszközöket 15 különbözõ csoportba lehet szervezni. Itt a lehetséges csoport számok 1 tól 15-ig vannak. Nézd meg a Send to beállítást lejebb is.
  • Port:
    Ha az MSC-t hálózaton keresztül küldöd, akkor szükséges beállítani egy IP portot(kaput) is. Ezt itt lehet megtenni. Az alapérték 6004. A port számának 6000 és 6100 között kell lennie.
  • Mód:
    Ide bökve megnyílik at MSC mód választó ablakocska:
    Három opció van itt:
    • Disabled - Ez ugyanaz mintha kikapcsolod az MSC bemenetet vagy kimenetet.
    • Ethernet - Ez az MSc-t hálózaton keresztül használja.This will use MSC via Ethernet - Itt találsz további olvasnivalót róla MSC és hálózat.
    • MIDI - Ez a pult MIDI csatlakozóját használja az MSC küldéséhez/fogadásához.
  • Exec:
    Bökj ide az MSC Exec ablakocskához:
    Három opció van itt:
    • Csak alapértelmezett - Ez az opció az MSC parancsokat csak a Master állomáson, a kiválasztott executorok felé irányítja, és onnan küldi el.
    • Exec.Page - Ezzel a parancsokat, csak egy kiváalsztott executorra lehet küldeni. Az oldal executor számát el kell váalsztani egy ponttal (Hex = 2E).
    • Exec Page - Ezzel a parancsokat, csak egy kiváalsztott executorra lehet küldeni. Az oldal executor számát el kell váalsztani egy ponttal (Hex = 2E).
  • Command:
    Ide bökve megnyílik a Válassz MSC Parancsot ablakocska:
    Három opció van itt:
    • Moving Light - Ez az opció a mozgófejes lámpák parancsmódját állítja be (hex 02).
    • General Light - Ez az opció az általános lámpák parancsmódját állítja be (hex 01).
    • All - Ez az opció minden típus formátumát állítja be (hex 7F).

 

A negyedik sorban van egy gomb Send tonéven. Ide bökve megnyílik aSelect Send to ablakocska ami így fest:

Három opció van itt:

  • Group:
    Ezzel szûkíthetõ az MSC átvitel a megadott csoportokra (1 - 15).
  • Device:
    Csak a meghatározott eszközre küld MSC-t (0 - 111).
  • All:
    Ez a beállítás Minden csatlakozott eszköznek továbbítja az MSC-t.

 

az MSC koncepciója

Az MSC parancsok az általános SysEx szerkezeten alapulnak, amit az MMA (MIDI Manufacturers Association) határozott meg. 1991-ben jelent meg és egy kiterjesztése a general MIDInek.

A nyers MIDI információkat hexadecimális oktettekbne lettek írva (két hexa szám). Különbözõ szoftvergyártók az MSC-t megpróbálták kiterjeszteni, emberek által olvashatóbb formára. Ez persze nagyon szép dolog, de mi nem tudjuk az összes variációt kifejteni most, ezért csak a nyers adatokra támaszkodunk.

Az üzenet formája így néz ki:

F0 7F Device ID 02 Command format Command Data F7
             
  • F0 7F ez a fejléc, ami azonosítja az üzenetet mint "universal system exclusive" és "real time" message.
  • Device ID ez az eszköz vagy csoportazonosító.
  • 02 ez egy hexa azonosító, hoy az üzenet MSC lesz.
  • Command Format ez egy okta szám ami meghatározza az eszköz típusát.
  • Command ez egy octa szám ami meghatározza a parancs típusát
  • Data ez az aktuális adat. Ez változhat a parancs típusától függõen.
  • F7 Ez egy lezáró oktettt, az üzenet vége jel.

Device ID

Az eszközazonosító(device ID) egy oktet ami három különbözõ részre van osztva:

  • 00 to 6F:
    Ez a 112 különbözõ eszköz azonosítószáma.
  • 70 to 7E:
    Ez a lehetséges 15 csoport száma.
  • 7F:
    Ez egy broadcast azonosító, amire minden eszköz hallgat.

A grandMA2-ben az eszköz és csoportazonosító lehet, mind a be illetve kimenõ üzenetekben.

Az MSC továbbításához csak egy oktettet továbbíthat az Eszköz-azonosító helyén. A Küldés beállításnak (fentebb leírt) kiválasztja a továbbítani kívánt három különféle azonosító egyikét. 

 

Parancs formátum(Command Format)

Az oktet parancsformátuma jelzi annak az eszköznek a típusát, amelynek meg kell kapnia az üzenetet (vagy legalább válaszolnia). A grandMA2 csak három különbözõ formátumban továbbítja és válaszol:

  • 01:
    Ez az General Lighting(általános világítás) fromátum
  • 02:
    Ez a Moving Lights(mozgófejes lámpa) formátum.
  • 7F:
    Ez az All(összes) formátum, amire minden eszköznek válaszolnia kellene.

 

Command(Parancs)

Az octet parancs az üzenetben szereplõ parancs típusát jelöli. A parancs típusa diktálja az adatinformációkat.

A grandMA2 7 különféle parancstípust támogat:

  • 01 (Go):
    Ez ugyanaz a parancs mint a Goto a grandMA2-ben. Ezt kötelezõ követnie egy kép számnak.
  • 02 (Stop):
    Ugyanaz mint a Pause parancs a pulton, utána egy kép számának kell lennie..
  • 03 (Resume):
    Ez "un-pause" egy képet. Ha egy bizonyos képet állítottunk le, akkor hasznos ha ide is beírjuk utána a kép számát :-).
  • 04 (Timed_Go):
    Ezzel lehet Goto parancsot adni megadott átúszási idõvel. Meg kell adniaz idõt és a kép számát is - ebben a sorrendben.
  • 06 (Set):
    Ez a húzók pozíciójának megadásához kell. Kell egy húzó szám és az oldal, majd a húzó pozíciója.
  • 07 (Fire):
    Ez makrók indításához kell. A makró számát kell a parancs után írni. Csak az 1 - 255-ig lévõ makrókat lehet indítani.
  • 0B (Go_Off):
    Ezzel lehet kikapcsolni a megadott executort. Ez után is meg kell adni a kép számát.

 

Data

01 (Go):

Mint fentebb kijelentettük: Ezt GOTO-parancsnak kell értelmezni. Meg kell adni egy kép számot. A zárószámot hexában is továbbítani kell, és a teljes számot hexákkal együtt. Ha a 4. számú képet kell aktiválni, akkor a teljes szám 4.000 - az összes tizedes szám ponttal elválasztott szám (Hex érték 2E). A tizedes számokat hexába a legegyszerûbb konvertálni. Elé kell írni egy "3"-ast. Ez azt jelenti, hogy a tizedes 4 34 lesz hexadecimális értékben. A teljes kép száma a ponttal együtt: 34 2E 30 30 30.

Lásd ezt a táblázatot a teljes decimál-hexadecimál fordításához.

Ha az állomást Default Only-ra állítottuk a Send MSC In Exec opcióban, akkor ezt mind hozzá kell adni a parancs után.

Például: beállítjuk a kép számát 21.5 -re az alapértelmezett executoron (All Devices, All Format, and Default Only beállításokkal):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 01 32 31 2E 35 30 30 F7

Ha a beállítások nem Default Only-ra vannak állítva, akkor az executort és az oldalt meg kell határozni. Két lehetõség van az executor és az oldalszám elkülönítésére. El lehet választani ponttal (hex = 2E) vagy szóközzel (hex = 20). A kép száma és az executor/page adatokat a 00hexa értékkel kell elválasztani egymástól.

Például: Beállítjuk a kép számát 37.2 az 5. executoron az elsõ oldalon (All Devices, All Format, és Exec.Page beállításokkal):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 01 33 37 2E 32 30 30 00 35 2E 31 F7

Ugyanaz a példa ezúttal szóközzel elválasztva az executor és az oldal:

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 01 33 37 2E 32 30 30 00 35 20 31 F7

 

02 (Stop(álj)):

Ez olyan mintha, megnyomnánák a pause gombot. Továbi információkért olvasd el a felsőbb fejezeteket a hexadecimáis kép számozás jobb megértése érdekében.

A Default Only opcióhoz nem kell semmilyen extra adatot megadni, mivel az executort szünetelteti.

Például: az alapértelmezett executor leállítása (All Devices, All Format, and Default Only beállításokkal):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 02   F7

Ha a pult küldi az MSC-t, akkor át fogja küldeni a kép számát 0.000. Ez olyan mintha elküldene egy “pause running cue” (állítsd meg a futó képet) parancsot.

Ha beállítások nincsenek Default Only-n, de a két Executor/Page opciók közül az egyik ki van választva akkor a 0. képet is továbítani kell. 

Példa: Megállítjuk a 5. executort az 1. oldalon (All Devices, All Format, and Exec.Page beállításokkal):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 02 30 2E 30 30 30 00 35 2E 31 F7

 

03 (Resume)(visszatérés):

Ez az egyetlen módja, hogy folyatassuk egy megállított kép futását. Az egyetlen különbség a Stop és a Resume parancsok között a "02" és "03".

Példa: Folytatjuk a kép félbehagyott úsztatását (All Devices, All Format, and Exec.Page beállításokkal):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 03 30 2E 30 30 30 00 35 2E 31 F7

Ha a csak alapértelmezett beálítás kivákasztva van akkor a parncso F0 7F 7F 02 7F 03 F7, ami kell az úszás folytatásához.

 

04 (Timed_Go(Időzített indulj)):

This is the same as the 01 (Go) command but with a specified time. Please read (and understand) about the 01 (GO) command and data above. Just to make this a bit simpler we are going to pretend that the “Default Only” option is turned on in the MSC options in Setup.

To transmit a timed Goto the time needs to be specified first and then the cue number. The time is specified by five hex octets. They represent (in order) Hour, Minute, Second, Frame, and Fraction.

The hour, minute, and second sections are very strait forward. The number needs to be transmitted in hex numbers. It is possible to transmit a value above the normal limit for example 64 seconds (hex = 40). The station will transmit this as 1 minute and 4 seconds.

Right now the console will not accept any time specified in the Frame and Fraction sections. But it transmit values below a second in the Frame section. The console divides the second into 24 frames. So 0.5 seconds is 12 Frames and the received hex number would be 0C.

Example: Goto cue 75 with the fade time of 20 seconds (All Devices, All Format, and Default Only setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 04 00 00 14 00 00 37 35 2E 30 30 30 F7

Example: Goto cue 5.4 with the fade time of 1 minute on executor 3 on page 1 (All Devices, All Format, and Exec.Page setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 04 00 01 00 00 00 35 2E 34 30 30 00 33 2E 31 F7

 

06 (Set):

The set command is used to move a fader to specific position. The 06 command is followed by two hex octets that indicates the fader and then two more octets that dictates the position.

 

The first of the two numbers for the fader is the fader number (on a page). The fader number 1 is hex number 00, the second is 01 and so on. Remember that this is a hex number so fader 16 have hex number 0F and decimal 17 is hex 10.

The second of the two numbers for the faders are the page number. This is a little different page 1 is hex number 01and page 2 is hex 02 and so on.

So executor 1 on page 1 is 00 01.

 

Calculating the position in hex numbers is a bit more tricky. The faders position is defined by a coarse and fine value. The scale for both values is 128 steps (most MIDI is in 128 steps). The fine value is transmitted first followed by the coarse value.

The desired fader position (in decimal) need to be multiplied by 1.28. The resulting integer is the coarse value. The remainder (everything on the right side of the separator) should be multiplied by 128 to get the fine value. The two decimal numbers then needs to be converted to hex.

Example:

The fader 3 on page 2 needs to be moved to 45%. First convert the position.

  1. Multiply 45 by 1.28 = 57.6
  2. Coarse value is 57.
  3. Multiply 0.6 by 128 = 76.8
  4. Fine value is 76.
  5. Convert decimal 57 to hex = 39.
  6. Convert decimal 76 to hex = 4C.
  7. Fine is transmitted before coarse so the position is: 4C 39

The MSC message is (All Devices and All Format setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 06 02 02 4C 39 F7

 

A fade time can be added after the fader and position data. The time format explained in the 04 command is used.

Example: Moving fader 15 on page 1 to 100% in 5 seconds (All Devices and All Format setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 06 0E 01 7F 7F 00 00 05 00 00 F7

 

The console only transmit the position of some faders - executors with sequences and all the green colored special masters (for the selected executor), but it accepts positions for all faders that have something assigned.

 

07 (Fire):

Macros can be fired by this command. It needs to be followed by a single octet specifying the macro number.

Macro number 1 is hex number 01. Hex number FF is macro number 255.

Example: Triggering macro number 64 (All Devices and All Format setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 07 40 F7

 

0B (Go_Off):

Executors can be turned off using the 0B command. A cue number needs to be transmitted - cue 0 can be used.

Example: Send an Off command to executor 9 on page 5 (All Devices, All Format, and Exec.Page setting):

F0 7F Device ID 02 Command Format Command Data F7
F0 7F 7F 02 7F 0B 30 2E 30 30 30 00 39 2E 35 F7

 

MIDI via Ethernet

MSC can sent using Ethernet. It is transmitted as a UDP message.

The MSC message is the same as described above, but the MSC message needs a header for the date to be accepted by the grandMA2.

The header separated into two parts. The first is identifying the message as a grandMA2 MSC message: 47 4D 41 00 4D 53 43 00 - it translates to GMA MSC.

The second part is four octets describing the length of the message - including the header. It is written as little endian byte format (least significant first).

This is calculated by counting the number of octets and then convert the decimal number into hex. Usually we only needs the first octet (it allows for messages up to 255 octets) although all four needs to be sent.

 

Examples:

Send a go command to cue 35. The MSC message is: F0 7F 7F 02 7F 01 33 35 2E 30 30 30 F7.

This is 13 octets. The header is always 12 octets. So the combined length is 25 octets. Decimal 25 is 18 in hex. The entire message is:

47 4D 41 00 4D 53 43 00 18 00 00 00 F0 7F 7F 02 7F 01 33 35 2E 30 30 30 F7

 

Trigger macro 1 is: F0 7F 7F 02 7F 07 01 F7 - 8 octets.

Plus the 12 from the header is decimal 20 = 13 hex.

The message is:

47 4D 41 00 4D 53 43 00 13 00 00 00 F0 7F 7F 02 7F 07 01 F7
